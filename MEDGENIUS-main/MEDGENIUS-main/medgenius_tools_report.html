1. Frontend Framework & Build Tools

1.1 React 19 (react + react-dom)

Version19.2.4
Packagereact, react-dom
TypeUI Library

What it is

React is a declarative, component-based JavaScript library for building user interfaces. Version 19 introduces automatic batching, concurrent rendering improvements, and streamlined ref handling.

Where it is used

React powers the entire frontend application. Every visual element the user sees is a React component.

FileHow React Is Used
main.jsxRoot ReactDOM.createRoot() render entry point
App.jsxRoot component — wraps in AuthProvider + ProgressProvider, state-based routing
AuthContext.jsxcreateContext, useContext, useState, useEffect for auth state
StudentProgressContext.jsxcreateContext, useCallback, useRef for debounced progress saves
16 page files in src/pages/Functional components with hooks
7 component files in src/components/Reusable shared UI components

Why it was chosen

  • Component-based architecture — each feature (Notes, Quiz, Flashcards, CDSS) is an independent component
  • Hooks API — useState, useEffect, useCallback, useRef, useContext simplify state management
  • Massive ecosystem — Firebase SDK, charting libraries, PDF tools all have React bindings
  • Performance — React 19's automatic batching prevents unnecessary re-renders

1.2 Vite 7 (Build Tool & Dev Server)

Version7.3.1
Packagevite
TypeBuild Tool + Dev Server

What it is

Vite is a next-generation frontend build tool using native ES modules for lightning-fast startup and HMR.

Where it is used

FileHow Vite Is Used

package.jsonnpm run dev → vite, npm run build → vite build

// vite.config.js
export default defineConfig({
  plugins: [react()],
  server: { port: 5173, open: true },

Why it was chosen

  • Speed — Dev server starts in < 300ms vs 10+ seconds with Webpack
  • Native ES Modules — No bundling during development
  • Environment variables — import.meta.env.VITE_GROQ_API_KEY keeps keys safe
  • Optimized production build — Rollup under the hood for tree-shaking & minification

Official Vite plugin for React. Adds JSX transform and React Fast Refresh. Referenced in vite.config.js as plugins: [react()].

2. UI & Styling

2.1 Vanilla CSS with CSS Custom Properties

Fileindex.css (48,637 bytes)
TypePure CSS — no framework

What it is

The entire application styling uses hand-written CSS with CSS Custom Properties for a cohesive dark-theme design system.

/* CSS Variables for theming */
--bg-primary: #0a0f1a;
--accent-primary: #00e6b4;
--glass-bg: rgba(255, 255, 255, 0.05);

/* Glassmorphism */
backdrop-filter: blur(12px);
background: var(--glass-bg);

Why vanilla CSS was chosen

  • Full control — No framework abstractions; every pixel is intentional
  • Performance — No runtime CSS-in-JS overhead; single file cached by browser
  • Dark medical theme — Custom dark mode with mint-green accents for reduced eye strain
  • Glassmorphism — Premium frosted-glass UI effects throughout

3. Backend Framework & Server

3.1 FastAPI (Python Web Framework)

Packagefastapi
TypeAsync Python Web Framework

What it is

FastAPI is a modern, high-performance Python web framework built on Starlette (ASGI) and Pydantic for automatic request validation and OpenAPI documentation.

Where it is used

FileHow FastAPI Is Used
main.pyCreates FastAPI() app, adds CORS middleware, registers routers
routers/lecture.pyAPIRouter for lecture processing endpoints
routers/anatomy.pyAPIRouter for /api/anatomy/generate-image

Features used

  • APIRouter — Modular route grouping (lecture, anatomy)
  • BaseModel (Pydantic) — Request body validation
  • StreamingResponse — SSE for real-time job progress
  • UploadFile — Multipart file upload handling
  • Auto-generated Swagger docs at /docs

Why it was chosen

  • Async support — Handles SSE streaming and concurrent requests
  • Type safety — Pydantic auto-validates request bodies
  • Auto-documentation — Free Swagger UI for API testing
  • Performance — One of the fastest Python frameworks

5. AI / LLM Providers

5.1 Groq Cloud API (Primary AI Provider)

API URLhttps://api.groq.com/openai/v1/chat/completions
Text Modelllama-3.3-70b-versatile (primary), llama-3.1-8b-instant, gemma2-9b-it

6. Document Parsing Libraries

6.1 pdfjs-dist (PDF Parsing)

Version5.4.624
Packagepdfjs-dist

Mozilla's PDF.js — extracts text from PDFs directly in the browser. Used in fileParser.js → parsePdfFile().

const pdfjsLib = await import('pdfjs-dist');
const pdf = await pdfjsLib.getDocument({ data: new Uint8Array(arrayBuffer) }).promise;
for (let i = 1; i <= pdf.numPages; i++) {
    const page = await pdf.getPage(i);
    const content = await page.getTextContent();
    fullText += content.items.map(item => item.str).join(' ');
}
  • Client-side — No server upload needed
  • Web Worker — Parsing in a background thread (UI stays responsive)
  • CDN fallback — Auto-loads from jsdelivr if local worker fails

6.2 mammoth.js (DOCX Parsing)

Version1.11.0
Packagemammoth

Extracts raw text from Word (.docx) files in the browser. Used in fileParser.js → parseDocxFile().

const mammoth = await import('mammoth');
const result = await mammoth.extractRawText({ arrayBuffer });
return result.value;

6.3 FileReader API (Text / PPTX)

Built-in browser API — handles .txt, .md, .csv files and basic PPTX XML extraction in fileParser.js. No extra library needed.

9. HTTP & Networking

9.1 Fetch API (Browser Built-in)

Every frontend AI service call uses native fetch(). Files: aiService.js, geminiService.js, translationService.js. Chosen over Axios for zero bundle size.

9.2 httpx (Python Async HTTP)

Used in anatomy.py for async HTTP calls to HuggingFace image generation API with 120-second timeout. Works with FastAPI's async def endpoints.

9.3 requests (Python Sync HTTP)

Used in ollama_service.py for synchronous calls to local Ollama LLM. 300-second timeout for long generation calls.

9.4 CORS Middleware

FastAPI middleware in main.py allowing frontend (port 5173) to call backend (port 8000).

10.1 React Context API

ContextFileManages

Chosen over Redux/Zustand because only 2 global state slices exist — Context is sufficient and zero-dependency.

10.2 localStorage (Offline Cache)

Caches progress data for sub-millisecond reads. Firestore writes are debounced (2s delay) to reduce network calls.

// Instant write to localStorage
localStorage.setItem(`medgenius_progress_${user.id}`, JSON.stringify(data));

// Debounced Firestore write (2s)
clearTimeout(saveTimerRef.current);
saveTimerRef.current = setTimeout(() => saveProgress(user.id, data), 2000);
